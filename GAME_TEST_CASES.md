## 前置条件

1. 将 `aimod-1.0.0.jar` 放入 `.minecraft/mods/` 目录
2. 使用 NeoForge 1.21.1 加载器启动游戏
3. 创建一个新的超平坦世界（创造模式，方便测试）
4. 在 `config/aimod-common.toml` 中配置 `apiKey`（若要测试 LLM 功能）

## 一、基础功能测试

### TC-01 生成 Bot

| 项目 | 内容 |
|------|------|
| **前置** | 进入一个存档，创造模式 |
| **操作** | 在聊天栏输入 `/ai_bot spawn` |
| **预期** | 1. 聊天栏显示 Bot 生成成功的消息<br>2. 玩家附近出现一个 Bot 实体（玩家模型外观）<br>3. Bot 站在玩家旁边，有生命值条 |

| 项目 | 内容 |
|------|------|
| **前置** | 已生成 Bot |

| **预期** | 1. 物品在地面上消失<br>2. DevLog 输出 `PICKUP_ITEM` 日志<br>3. 物品进入 Bot 背包 |
| **验证** | 给 Bot 分配任务让它打开背包可间接验证 |

### TC-03 生成多个 Bot

## 三、LLM 智能任务测试

> 需要在 `config/aimod-common.toml` 中配置有效的 `apiKey`

### TC-15 自然语言指令 — 挖矿

| 项目 | 内容 |
|------|------|
| **前置** | 已配置 API Key，Bot 附近有钻石矿 |
| **操作** | `/ai_bot task 帮我挖 5 个钻石矿` |
| **预期** | 1. Bot 调用 LLM API 解析指令<br>2. LLM 返回行动序列（mine + say）<br>3. Bot 搜索并挖掘钻石矿<br>4. 完成后 Bot 聊天广播完成消息 |
| **验证** | DevLog 输出 `LLM_REQUEST` → `LLM_RESPONSE` → `TASK_PARSE_DONE` 日志链 |

### TC-16 自然语言指令 — 收集木材

| 项目 | 内容 |
|------|------|
| **前置** | Bot 附近有树木 |
| **操作** | `/ai_bot task 去砍 10 块木头` |
| **预期** | 1. Bot 解析为 GatherResourceAction(WOOD, 10)<br>2. Bot 走到最近的树前砍伐<br>3. 木头进入 Bot 背包 |

### TC-17 自然语言指令 — 复合任务

| 项目 | 内容 |
|------|------|

### TC-18 无 API Key 时的 Fallback

| 项目 | 内容 |
|------|------|
| **前置** | 确保 `apiKey` 为空 |

## 已知限制

- **无持久化**：Bot 背包和任务不会在服务器重启后保存
- **WorldScanner 性能**：大范围扫描（radius>10）可能导致卡顿
- **HttpClient 未复用**：每次 LLM 调用创建新连接
- **线程安全**：LLMService 健康检查缓存存在竞态条件风险
- **语言文件**：zh_cn.json 存在编码问题（mojibake）
